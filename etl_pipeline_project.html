Healthcare Data ETL Pipeline

SQL-Based Data Integration Solution

A robust and scalable ETL (Extract, Transform, Load) pipeline designed to process and integrate large healthcare datasets from multiple sources. This solution creates a unified data warehouse that supports advanced analytics and machine learning models for hospital readmission prediction and healthcare insights.

Project Overview

Healthcare organizations generate vast amounts of data from various sources including Electronic Health Records (EHR), laboratory systems, imaging systems, and administrative databases. This ETL pipeline consolidates these disparate data sources into a unified, clean, and analysis-ready dataset that serves as the foundation for predictive modeling and business intelligence.

Technical Implementation

SQL-Based Processing: Leveraged advanced SQL techniques including stored procedures, views, and functions for efficient data processing and transformation.

-- Sample SQL for patient readmission data processing CREATE PROCEDURE ProcessReadmissionData AS BEGIN -- Clean and standardize patient data UPDATE PatientRecords SET diagnosis_code = UPPER(TRIM(diagnosis_code)) WHERE diagnosis_code IS NOT NULL; -- Calculate readmission indicators WITH ReadmissionCTE AS ( SELECT patient_id, admission_date, discharge_date, LEAD(admission_date) OVER (PARTITION BY patient_id ORDER BY admission_date) as next_admission FROM Admissions ) UPDATE Admissions SET readmission_30day = CASE WHEN DATEDIFF(day, discharge_date, next_admission) <= 30 THEN 1 ELSE 0 END; END

Python Integration: Used Python with Pandas and SQLAlchemy for complex data transformations and automated data quality reporting.
